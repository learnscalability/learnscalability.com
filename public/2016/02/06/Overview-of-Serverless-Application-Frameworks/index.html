Overview of Serverless Application Frameworks

This is a growing document on technologies people use to build serverless websites.

Available Frameworks

  • serverless/serverless Built on nodejs and formarly know as JAWS, this framework appears to be the most mature, stable and feature full of all those surveyed.
  • MitocGroup/deep-framework Developed in nodejs, plenty of features, but with much less adoption.
  • mentum/lambdaws Built in nodejs, this framework is not intended for setting up Lambda powered APIs, it doesnâ€™t have integration with API Gateway. Instead it features SQS support and is intended for offline processing.
  • apex/apex. Written in golang, by the famous TJ Holowaychuck, this framework is popular and stable.
  • exratione/lambda-complex. A collection of nodejs-based tools for developing for AWS Lambda.
  • tim-b/grunt-aws-lambda. Collection of grunt.js tasks for managing lambda functions in AWS

Alternatives

  • Iron.io Worker supports multiple languages, and offers a containerized environment.
  • Tutum.co is a platform for running container on demand, they got aquired by docker.
  • StackHut.com is a github project, provides similar usage to AWS lambda, only you choose the hosting.
  • hook.io lets you hook a gist script to a webhook, is open source and is built by the awsome Marak Squires of Nodejitsu fame.
  • webscript.io cloud workers, handling strictly lua functions.
